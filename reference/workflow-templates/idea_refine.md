# スキル: アイデア精錬 (Idea Refine)

## 目的
生のアイデアを構造化された発散・収束プロセスで磨き、実行可能な提案に変換する。「なんとなく良さそう」を防ぎ、前提・リスク・やらないことを明示する。

## いつ使うか
- 新しいビジネスアイデア・施策・仮説を検討するとき
- 「何をすべきか」が複数あり優先度を絞りたいとき
- 企画書・提案書を作成する前の思考整理

## 手順

### フェーズ 1：展開（発散）
1. アイデアを「どうすれば〜できるか（How Might We）」の問いとして言い換える
2. 以下の観点から 5〜8 つのバリエーションを生成する
   - 逆転：真逆の前提に立ったらどうなるか
   - 制約除去：コスト・時間・人員の制限がなければ
   - 最小化：最小の手間で成立する形は
   - 組み合わせ：別の手段・市場・顧客と組み合わせたら
3. 誰のどの課題を解くか（ターゲット）を明確にする

### フェーズ 2：評価（収束）
4. バリエーションを 2〜3 方向にグループ化する
5. 各方向を以下の 3 軸で評価する
   - ユーザー価値：誰が・なぜ・どれだけ嬉しいか
   - 実現可能性：今の自社リソースで着手できるか
   - 差別性：競合・代替手段との違いは何か
6. 隠れた前提と致命的リスク（Kill Risk）を明示する

### フェーズ 3：一枚紙化（シャープニング）
7. 以下の構成で提案一枚紙を作成して `output/` に保存する
   ```
   問題定義（Problem Statement）
   推奨方向と選択理由
   検証すべき主要前提
   MVP スコープ（最小の着手内容）
   やらないこと（Not Doing）とその理由
   ```

## アンチパターン
- 20 個以上のアイデアを並べて「あとはオーナーが選ぶ」とする
- 「誰向けか」を決めずに施策を提案する
- Kill Risk（致命的前提の崩壊）を検討しない

## 注意事項
- "Not Doing" リストは提案と同じくらい重要。省略しない
- 一枚紙は `agents/{自分}/output/` に保存し、ログに記録する
