# スキル: 自己レビュー (Self Review)

## 目的
各エージェントが自身のログ・rules.json・skills を週次で点検し、改善提案を構造化された形で残す。「気づいたら忘れる」を防ぎ、ルールとスキルを自律的に成長させる。

## いつ使うか
- 毎週金曜のセッション末（営業日でない場合は直前の営業日）
- 大きなタスク完了直後（締切ベースの振り返り）
- 同じ問題に 2 回以上ぶつかったとき（再発防止のためのルール化検討）

## 入力
- `agents/{自分}/logs/current.jsonl` の直近 1 週間分
- `agents/{自分}/rules.json`
- `agents/{自分}/CLAUDE.md`
- `agents/{自分}/skills/` 一覧
- `Agent-team/reviews/peer/` のうち、自分宛て（`*_to_{自分}.md`）の未対応レビュー

## 手順

### 1. 事実収集（読み取りのみ）
1. `logs/current.jsonl` から直近 7 営業日のエントリを抽出する
2. `status` 別に分類する（done / in_progress / blocked）
3. `issues` フィールドを全て列挙する（問題の頻度をカウント）
4. `decisions` フィールドから今週採用した判断を列挙する

### 2. 5 つの観点で評価
各項目に対して「良かった点 / 改善点 / 次の打ち手」を 1〜2 行で記述する。

| 観点 | 問い |
|------|------|
| アウトプット品質 | 出力した成果物はオーナーの期待値を超えたか |
| 判断速度 | blocked が長引いた案件はあるか・原因は何か |
| ルール充足度 | rules.json で曖昧だった or 不足していた項目はあるか |
| スキル化機会 | 2 回以上手で繰り返した手順はあったか |
| コンテキスト効率 | 不要な情報を読み込んでいなかったか・必要な情報を見落とさなかったか |

### 3. 改善アクションの起票
評価結果から、次の 3 カテゴリのいずれかに振り分ける。

- **rules.json 更新**: 判断基準・禁止事項・好みを追記/修正する
- **skills/ 追加 or 更新**: 繰り返し手順をスキル化する → 新規 .md を `agents/{自分}/skills/` に作成
- **オーナー確認事項**: 自己判断で決められない事項を `open_questions` に追加し、次回オーナーへ提示

### 4. レポート出力
`Agent-team/reviews/self/{agent}/YYYY-MM-DD.md` を以下のテンプレートで作成する。

```markdown
# {agent} 自己レビュー — YYYY-MM-DD

## サマリー
- 対象期間: YYYY-MM-DD 〜 YYYY-MM-DD
- 完了タスク数: N
- 進行中: N / ブロック: N
- 主要な意思決定: {1〜3 行}

## 5 観点評価
| 観点 | 良かった点 | 改善点 | 次の打ち手 |
|------|----------|--------|----------|
| アウトプット品質 | … | … | … |
| 判断速度 | … | … | … |
| ルール充足度 | … | … | … |
| スキル化機会 | … | … | … |
| コンテキスト効率 | … | … | … |

## 改善アクション
- [ ] rules.json 更新: {変更内容} — 担当: 自分・期限: YYYY-MM-DD
- [ ] skills/ 追加: {スキル名} — 担当: 自分・期限: YYYY-MM-DD
- [ ] オーナー確認: {open_question} — 次回セッションで提示

## 受領済み相互レビュー（peer）への対応状況
- {peer review file path}: {対応済み / 対応中 / 却下＋理由}
```

### 5. ログ・コミット
1. `logs/current.jsonl` に追記する
   - `task` は `"self_review"`、`summary` に「{N} 件の改善アクションを起票」
   - `decisions` に rules.json 更新内容があれば併記
2. `git push` でコミット&push する

## アンチパターン
- 「特に問題なし」で終わる（5 観点全てを埋めることが必須）
- 改善アクションを起票するだけで実装しない（次週の自己レビューで未消化を必ず確認する）
- 自己レビューを skip する（隔週でも実施し、その旨レポートに明記する）

## 注意事項
- レポートは 200〜400 行を上限とする（読まれないと意味がない）
- 同じ改善アクションが 3 回連続で先送りされたら、`open_questions` に上げてオーナー判断を仰ぐ
- 出力先 `reviews/self/{agent}/` は git 管理対象（gitignore で除外しない）
